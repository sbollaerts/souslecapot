# Procédure en cas de trajet resté ouvert après restitution

Cette procédure décrit le traitement d’un trajet qui apparaît encore actif après que le membre affirme avoir restitué le vélo.

## Symptômes fréquents

Le membre peut signaler l’une des situations suivantes :

- l’application affiche toujours un trajet en cours ;
- le compteur continue d’augmenter ;
- le vélo a été placé dans une station ou une zone autorisée, mais la fin de trajet n’a pas été confirmée ;
- un montant anormalement élevé apparaît dans l’application ;
- le membre a entendu le signal de verrouillage, mais n’a pas reçu de confirmation.

## Première action demandée au membre

Le membre doit d’abord vérifier dans l’application si le trajet peut encore être clôturé manuellement.

Si le vélo est toujours accessible, il doit vérifier que :

1. le vélo est correctement immobilisé ;
2. le verrou est complètement engagé ;
3. le vélo se trouve dans une zone de stationnement autorisée ;
4. l’application dispose d’une connexion réseau ;
5. la tentative de fin de trajet a été relancée une fois.

Le membre ne doit pas multiplier les tentatives si le verrou semble endommagé.

## Informations à collecter

Pour ouvrir une demande de révision, le service à la clientèle collecte :

- l’identifiant du membre ;
- l’identifiant du trajet, s’il est visible ;
- l’heure approximative de restitution ;
- l’emplacement de restitution ;
- une photographie du vélo ou de son environnement, si disponible ;
- une description du message affiché dans l’application.

## Traitement par le service à la clientèle

Le service à la clientèle :

1. vérifie que le membre est correctement identifié ;
2. consulte le statut opérationnel du trajet ;
3. vérifie le dernier emplacement connu du vélo ;
4. compare l’heure déclarée avec les événements techniques disponibles ;
5. détermine si une demande de révision doit être créée ;
6. informe le membre qu’aucun remboursement n’est garanti avant analyse.

Le document décrit le processus documentaire. Le statut réel du trajet doit être consulté dans le système opérationnel et ne peut pas être déduit de cette procédure.

## Cas nécessitant une escalade

Une escalade est requise lorsque :

- le vélo est introuvable ;
- plusieurs trajets présentent le même problème sur une courte période ;
- le verrou ou le mécanisme de restitution semble défectueux ;
- les données techniques sont contradictoires ;
- le montant contesté dépasse 75 € ;
- un risque de fraude ou de sécurité est suspecté.

## Délai de traitement

Une demande complète est généralement examinée dans un délai de trois jours ouvrables.

Ce délai est indicatif. Il ne constitue pas une garantie contractuelle.

## Réponse à donner

Le service à la clientèle peut confirmer qu’une demande a été enregistrée, mais ne doit pas annoncer un remboursement avant la fin de l’analyse.
